# 访谈稿件编码分析工具 · 使用指南

## 📌 工具简介
这是一个基于 DeepSeek 大模型的**访谈稿件编码分析工具**。你只需要按照界面引导填写研究背景、访谈提纲和多用户的访谈原文，工具就会自动生成分析报告，包括：
- 按提纲维度汇总的核心结论
- 引用自用户的原声证据
- 每位受访者的简短特征画像（100 字内）
- 提纲之外的额外发现（如果开启）

工具提供了 **两种分析方式**，你可以根据需要任选其一：
1. **自动调用 API**：输入 DeepSeek API Key，一键发送并自动解析结果。
2. **手动粘贴 JSON**：自行调用大模型获取 JSON ，粘贴到工具中解析。

整个过程由 4 张卡片引导完成，我们逐张来看。

---

## 🧩 第一张卡片：研究定向 & 提纲层级解析

这里是配置研究的「基座」，决定 AI 将以什么框架来分析访谈内容。

### ① 两种填写模式
- **✍️ 单块输入 (默认)**：逐项填写，适合当场编写。
- **📦 整体一键输入**：将预先准备好的文本整体粘贴或上传，工具自动拆分填充。文本中需要包含以下标记：
```
【访谈主题】
【背景信息】
【研究目标】
【提纲维度】
【重点关注】
【自定义指令】
```
### ② 字段说明
| 字段 | 是否必填 | 作用 |
|---|---|---|
| 📌 访谈主题 | 选填 | 告诉 AI 研究主题 |
| 📄 背景信息 | 选填 | 项目相关背景，帮助 AI 理解上下文 |
| 🎯 研究目标 | 选填 | 核心问题或研究目标 |
| 📋 **提纲维度** | **必填** | 分析框架，支持多级编号（如 1. / 1.1 / ① 等），工具会自动解析为树形结构 |
| ⭐ 重点关注 | 选填 | 希望 AI 特别留意的内容，相关原声会被优先提取 |
| ✨ 允许额外发现 | 可选 | 勾选后，AI 会输出提纲之外的发现维度 |
| ✏️ 自定义指令 | 选填 | 额外要求，如“每个维度至少总结 2 个核心观点” |

> 💡 提纲维度示例：
> ```
> 1. 使用习惯
> 1.1 使用频率
> 1.2 使用场景
> 2. 痛点分析
> ```

填写提纲后，点击「**✅ 保存研究定向 & 解析提纲**」，下方会显示识别出的层级树，确认无误即可进入下一步。

### ③ 批量输入技巧
切换到「📦 整体一键输入」标签，可以：
- 选择本地 `.txt` 或 `.md` 文件上传
- 或直接在文本框中粘贴按标记组织好的内容

点击「**📝 解析并填充到表单**」，会自动把内容填充到单块输入区，此时再回到单块标签检查并保存。

## 👥 第二张卡片：多用户访谈文本

这里添加每个受访者的姓名和对应的访谈转录稿。

### ① 默认已有一名受访者
工具会事先创建一个空的受访者卡片，你只需：
- 修改**受访者名称**
- 在文本框中粘贴或输入该用户的访谈记录

### ② 新增/删除/上传
- 点击「**+ 添加受访者**」增加用户
- 每张用户卡片上都有「📄 上传文件」按钮，可直接上传 `.txt` / `.md` 文件
- 不需要的用户可点击「删除」按钮移除

### ③ 保存
所有文本修改会自动暂存，点击「**✅ 保存所有用户**」可以手动确认（实际上用户数据会实时更新，但保存时会有提示）。

> ⚠️ 注意事项：**至少需要一位用户填写了内容**，否则后续按钮会禁用。

## 🤖 第三张卡片：AI 调用方式

这是核心操作区，提供两种分析途径。

### 方式一：一键自动调用（推荐）
如果你有 **DeepSeek API Key**：

1. 在卡片顶部的 **API 配置区** 输入 Key，勾选「记住 Key」可免去重复输入（存储于浏览器本地）。
2. 模型选择：`DeepSeek-V4-Flash`（通用推荐）或 `DeepSeek-V4-Pro`（思考链）。
3. 点击「**🚀 一键发送至DeepSeek并自动解析**」。
4. 等待几秒至几十秒（视访谈文本长度），状态栏会提示「✅ AI 分析完成」，第四张卡片会直接展示结果。

> 🔑 如何获取 API Key：前往 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册并创建 Key。

### 方式二：手动粘贴 JSON（备选）
如果你希望使用其他大模型，或手动将 Prompt 发给 DeepSeek 网页版：

1. 点击「**✨ 生成完整Prompt并复制**」——工具会自动构建包含所有背景、提纲和访谈文本的完整 Prompt 并**一键复制到剪贴板**，左上角会弹出复制成功提示。
2. 将这段文本发送给 AI（例如 ChatGPT、网页版 DeepSeek 等），并获取返回的 JSON。
3. 回到工具，在第三张卡片底部找到折叠的「**📋 手动粘贴AI返回的JSON结果（备选）**」，点击展开。
4. 将获得的 JSON 粘贴入文本框。
5. 点击「**🔍 解析并展示**」，同样生成报告。

> 🧩 两种方式生成的结果完全互通，都支持导出。

## 📊 第四张卡片：AI 结果报告

当 AI 分析完成或手动 JSON 解析成功后，这里会展示最终报告，包含三部分：

### 👤 受访者画像小结
工具为每个受访者自动生成一句 100 字以内的概括性描述，提炼其关键特征、行为或态度。

### 📑 提纲维度洞察
以树形结构展示你设置的提纲每个维度的：
- **核心结论**（综合所有受访者）
- **用户原声**（带用户标签的原文引用，可展开/折叠）

点击节点可折叠/展开，方便快速浏览重点。

### 提纲外额外发现
如果勾选了「允许 AI 补充额外发现维度」且 AI 确实有输出，会在此独立展示。

### 📑 导出 Markdown 报告
点击右下角「**📑 导出Markdown报告**」，生成一个完整的 `.md` 文件，包含以上所有内容，可直接存档或分享给团队成员。

## 🧰 常见问题

**Q：点击按钮后没反应？**  
A：检查①提纲维度是否填写并保存；②是否至少有一名用户填写了访谈内容；③API Key 是否正确（自动模式）。

**Q：手动粘贴 JSON 解析失败？**  
A：请确认粘贴的是完整的 JSON 对象，包含 `outline_insights`、`user_profiles` 等字段，且格式合法。可以尝试去掉 JSON 前后的代码块标记（```）。

**Q：能否中途修改提纲或用户文本？**  
A：可以，修改后需要重新生成 Prompt 并再次调用 AI（或重新粘贴 JSON），结果会覆盖之前的内容。

**Q：工具会保存我的数据吗？**  
A：所有数据仅在浏览器中暂存，刷新页面后会丢失（API Key 勾选记住后保存在本地）。建议及时导出报告保存。

## ✅ 标准工作流总结

1. 填写研究背景和**提纲维度**（必填）。
2. 添加受访者，输入/上传访谈转录稿。
3. 选择自动调用或手动粘贴：
 - 自动：输入 API Key → 点击「一键发送」。
 - 手动：生成 Prompt 并复制 → 自行调 AI → 粘贴 JSON 解析。
4. 查看报告，可折叠浏览、展开原声。
5. 导出 Markdown 报告。
